利用Mabatis逆向工程快速搭建SSM框架

发表于2021-06-22|更新于2022-05-21|SpringBoot
|字数总计:1.4k|阅读时长:5分钟|阅读量:

准备工作

为方便测试，在本地数据库准备好一张表
以及一个JDBC驱动jar包

正式开始

上面两项工作做好之后，就可以正式开始了

逆向工程创建

  1. 创建一个简单的springboot项目
    相关配置注意jdk设为8就行

    注意勾选为Spring Web项目

  2. 创建完之后，就是相关的配置

    • 添加MySQL驱动依赖和Mybatis整合SpringBoot框架的起步依赖
    • 然后是在项目根目录下添加GeneratorMapper.xml文件

      注意：把上面JDBC驱动包的位置设置为你本地的路径，数据库连接信息也改为你的，targetPackage的包结构也要对应起来,table标签一张表对应一个实体类和其对应的dao接口和映射文件，这里只用到一张表所以只写了一个，如果有多张表那么则需要写多个table标签

    • 然后还需要在pom文件中添加逆向工程需要的插件 添加完之后记得reimport不然读取不到pom中添加的依赖

      刷新之后可以看到项目插件中多了mybatis-generator，双击就可以看到自动生成了实体类、Dao接口和映射文件
      至此，逆向工程创建完成了~

控制层、业务层的创建

逆向工程创建好，就等于持久层创建完了，现在只需要创建控制层和业务层即可

  1. 控制层，这里创建一个StudentController
  2. 业务层，需要创建控制层调用的接口和对应的实现类，并加上service注解
  3. Dao层的接口虽然逆向工程的时候自动为我们添加了方法，但在业务层调用的时候需要使用autowired注解从spring容器中获取，所以需要添加注解Mapper

    核心配置文件的配置

    上面配置好之后，还需要在核心配置文件中(application.properties)添加必要的数据库配置

    手动添加指定文件夹

    上面的步骤以及差不多了，但是我们测试的时候出现报错，原来是dao层中的映射文件没被编译，这里有两种解决方案
  4. 在pom中手动指定资源文件夹，使映射文件可以被编译（同样需要reimport）
  5. 将映射文件移动到resources资源文件夹,并在核心配置文件中声明位置

成功测试

启动项目，在浏览器上测试
可以看到返回到了数据，大功告成~

补充

  1. 上面在业务层使用Autowired注解调用Dao层，除了添加mapper注解之外，还可以在核心配置文件中添加MapperSacn注解，其值为dao接口所在包
  2. Mybatis逆向生成只针对单表，并且数据库中的字段由多个单词构成时必须用_分隔
